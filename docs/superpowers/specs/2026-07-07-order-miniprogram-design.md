# 情侣点餐小程序 — 设计文档

> 日期：2026-07-07 | 状态：已确认，待实现

## 一、项目概述

一款微信小程序，女友端可以像点外卖一样"下单"奶茶/零食/餐食，选择规格和备注，提交后通过微信订阅消息推送到男友微信。男友可在小程序中管理菜单、处理订单、更新状态，完成后女友也会收到状态通知。全程免费，基于微信云开发实现。

## 二、核心决策

| 决策项 | 选择 | 原由 |
|--------|------|------|
| 通知方式 | 微信订阅消息 | 官方支持、稳定、免费、双向通知 |
| 菜单管理 | 管理员管理 + 用户可提建议 | 灵活且有互动性 |
| 下单模式 | 详细模式 | 支持多规格选择+备注，体验真实 |
| 身份认证 | wx.login + openid | 免费、无缝、安全 |
| 订单流程 | 双向通知 | 下单通知 → 处理 → 完成回执 |
| 后端 | 微信云开发 | 免费配额够用、零运维、免备案 |

## 四、数据模型

### 4.1 users 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `_openid` | string | 微信自动生成，唯一标识用户 |
| `nickName` | string | 微信昵称 |
| `avatarUrl` | string | 微信头像 |
| `role` | string | `"admin"` 管理员 / `"user"` 普通用户 |
| `createdAt` | date | 创建时间 |

> 管理员设置方式：在云开发控制台手动将目标用户 `role` 改为 `"admin"`。

### 4.2 menus 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 菜品名称 |
| `category` | string | `"drink"` / `"snack"` / `"meal"` |
| `image` | string | 云存储 fileID |
| `price` | number | 参考价格（可选） |
| `specs` | array | 规格配置 |
| `available` | boolean | 是否上架 |
| `createdAt` | date | 创建时间 |

`specs` 结构（每个菜品可自定义）：
```json
[
  { "name": "冰量", "options": ["正常冰", "少冰", "去冰", "常温", "热"] },
  { "name": "甜度", "options": ["全糖", "七分糖", "半糖", "三分糖", "无糖"] },
  { "name": "杯型", "options": ["中杯", "大杯"] }
]
```

### 4.3 orders 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `_openid` | string | 下单用户的 openid |
| `items` | array | 订单项列表 |
| `status` | string | `"pending"` → `"accepted"` → `"preparing"` → `"done"` |
| `note` | string | 额外备注 |
| `createdAt` | date | 下单时间 |
| `updatedAt` | date | 最后状态更新时间 |

`items` 数组结构：
```json
[
  {
    "menuId": "xxx",
    "name": "珍珠奶茶",
    "quantity": 1,
    "specs": { "冰量": "少冰", "甜度": "半糖", "杯型": "大杯" },
    "remark": "多加珍珠"
  }
]
```

### 4.4 suggestions 集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `_openid` | string | 建议人的 openid |
| `name` | string | 建议的菜品名 |
| `category` | string | 建议的分类 |
| `reason` | string | 理由 |
| `status` | string | `"pending"` / `"approved"` / `"rejected"` |
| `createdAt` | date | 创建时间 |

## 五、页面结构

```
pages/
├── index/            首页 — 分类Tab + 菜品卡片 + 购物车入口
├── menu/             菜单浏览 — 分类筛选 + 列表
├── detail/           菜品详情 — 选规格 + 数量 + 备注 → 加入购物车
├── cart/             购物车 — 确认订单项 + 提交
├── orders/           我的订单 — 订单列表 + 状态
├── suggest/          建议菜品 — 提交建议
├── admin-menu/       管理员菜单管理 — 增删改
├── admin-orders/     管理员订单处理 — 查看 + 改状态
└── admin-suggest/    管理员建议审核
```

### 页面路由规则（app.json tabBar）

底部 tabBar（所有用户可见）：
- 首页 (index)
- 菜单 (menu)
- 购物车 (cart)
- 我的 (orders)

管理相关页面通过"我的"页面中的入口进入（仅 admin 可见）。

## 七、订阅消息设计

### 7.1 消息场景

| 场景 | 模板类型 | 接收人 | 触发时机 |
|------|---------|--------|---------|
| 新订单通知 | 新订单提醒 | 管理员 | 用户提交订单后 |
| 订单状态更新 | 订单状态变更 | 用户 | 管理员标记完成后 |

### 7.2 授权时机

- **用户端**：提交订单成功后弹出授权引导 — "需要订阅订单状态通知吗？完成后会第一时间告诉你~"（拒绝不影响下单）
- **管理员端**：首页/管理入口放置"开启接单通知"按钮，点击授权一次

### 7.3 消息示例

管理员收到：
> 【新订单】珍珠奶茶 x1（少冰·半糖·大杯）| 备注：多加珍珠
> 点击查看详情 →

用户收到：
> 【订单更新】你的珍珠奶茶已经在路上了~
> 当前状态：已完成 | 点击查看详情 →

## 八、云函数

| 云函数 | 功能 | 权限 |
|--------|------|------|
| `login` | 获取 openid，查/建用户，返回 role | 无 |
| `getMenus` | 获取可用菜单列表 | 无 |
| `addMenu` | 添加菜品 | admin |
| `updateMenu` | 修改菜品 | admin |
| `deleteMenu` | 软删除菜品（设为不可用） | admin |
| `submitOrder` | 创建订单 + 触发管理员推送 | 无 |
| `getOrders` | 获取订单（admin 看全部，user 看自己） | 自动区分 |
| `updateOrderStatus` | 更新订单状态 + 触发用户推送 | admin |
| `submitSuggestion` | 提交菜品建议 | 无 |
| `getSuggestions` | 获取建议列表 | admin |
| `handleSuggestion` | 处理建议（通过则写入 menus） | admin |

## 十、购物车设计

- 使用 `wx.setStorageSync` 本地存储，无需服务端
- 数据结构：`[{ menuId, name, image, quantity, specs, remark }]`
- 右下角浮动图标 + 角标数量，始终可见
- 提交成功后清空本地购物车

## 十一、技术栈

| 层面 | 选择 | 说明 |
|------|------|------|
| 小程序框架 | 微信原生 | 轻量，无额外依赖 |
| 语言 | JavaScript ES6+ | 前后端统一 |
| 后端 | 微信云开发 | 云函数 + 云数据库 + 云存储 |
| 样式 | WXSS + Flex | 原生样式 |
| 图标 | Emoji / 本地图片 | 保持包体积小 |

## 十二、首页设计要点

- 顶部：问候语 / banner（可配置昵称）
- 分类 tab：🥤奶茶 / 🍿零食 / 🍱餐食
- 菜品区域：双列卡片网格，每张卡片显示图片 + 名称 + 可选价格
- 右下角：购物车浮动按钮 + 角标
- 左上角：用户头像 → 点击进入个人中心/管理入口

## 十三、管理入口设计

- 普通用户点击头像：显示"我的订单"+"建议菜品"
- 管理员点击头像：额外显示"菜单管理"+"订单管理"+"建议审核"
- 通过 `role === "admin"` 控制可见性
